# Порядок запуска

1. Запустить докер контейнер
    - Выполнить команду в папке с проектом: "**docker-compose up -d**"
2. Запустить модуль **infra/discovery-server**
3. Запустить модуль **infra/config-server**
4. Запустить модуль **infra/gateway-server**
5. Запустить модуль **stat/aggregator**
6. Запустить модуль **stat/analyzer**
7. Запустить модуль **stat/collector**
8. Запустить модуль **core/user-service**
9. Запустить модуль **core/event-service**
10. Запустить модуль **core/request-service**
11. Запустить модуль **core/rating-service**

# Местоположение файлов настроек проекта

Настройки используемые микросервисами получаются с помощью Config-server.
Местоположение всех настроек находится по адресу
~~~
/infra/config-server/src/main/resources/config/{application}/
~~~
Пример местоположения для **core/user-service**
~~~
/infra/config-server/src/main/resources/config/user-service/
~~~

# Зависимости модулей и краткое описание реализуемых эндпоинтов

## Микросервис User-service

### Реализуемые эндпоинты user-service

**POST:**
- **`/admin/users`** — Создание нового пользователя.
    - Принимает `NewUserRequest` в теле запроса.
    - Возвращает созданного пользователя со статусом `201 Created`.

**GET:**
- **`/admin/users`** — Получение списка пользователей с пагинацией.
    - Принимает параметры:
        - `ids` (опционально, список `Long`) — фильтрация по идентификаторам пользователей.
        - `from` (по умолчанию `0`) — смещение для пагинации.
        - `size` (по умолчанию `10`) — количество записей на страницу.
    - Возвращает список пользователей со статусом `200 OK`.

- **`/admin/users/allUsersById`** — Получение списка пользователей по их идентификаторам (без пагинации).
    - Принимает параметр:
        - `ids` (опционально, список `Long`) — идентификаторы пользователей.
    - Возвращает список пользователей со статусом `200 OK`.

- **`/admin/users/{userId}`** — Получение пользователя по его идентификатору.
    - Принимает параметр пути:
        - `userId` (`Long`) — идентификатор пользователя.
    - Возвращает пользователя со статусом `200 OK`.

**DELETE:**
- **`/admin/users/{userId}`** — Удаление пользователя по его идентификатору.
    - Принимает параметр пути:
        - `userId` (`Long`) — идентификатор пользователя.
    - Возвращает статус `204 No Content` без тела ответа.

## Микросервис request-service

### Зависимости от Feign-клиентов request-service

Контроллер использует следующие Feign-клиенты для взаимодействия с микросервисами:

**UserServiceFeign** (`user-service`):
- `GET /admin/users/{userId}` — получение пользователя по идентификатору

**EventServiceFeign** (`event-service`):
- `GET /events/{id}/WithoutHttp` — получение события по идентификатору

---

### Реализуемые эндпоинты request-service

**GET:**
- **`/users/{userId}/requests`** — Получение всех запросов на участие текущего пользователя.
    - Принимает параметр пути:
        - `userId` (`Long`) — идентификатор пользователя.
    - Возвращает список запросов на участие со статусом `200 OK`.
    - Зависит от `UserServiceFeign.getUser()` для проверки существования пользователя.

- **`/users/{userId}/requests/{eventId}/requests`** — Получение всех запросов на участие в событии пользователя.
    - Принимает параметры пути:
        - `userId` (`Long`) — идентификатор пользователя.
        - `eventId` (`Long`) — идентификатор события.
    - Возвращает список запросов на участие со статусом `200 OK`.
    - Зависит от `EventServiceFeign.getEventByIdWithoutHttp()` — для проверки существования события.

- **`/users/{userId}/requests/{eventId}?status={status}`** — Проверка регистрации пользователя на событие.
    - Принимает параметры:
        - `userId` (`Long`) — идентификатор пользователя.
        - `eventId` (`Long`) — идентификатор события.
        - `status` (`RequestStatus`) — статус запроса.
    - Возвращает `boolean` — подтверждение регистрации пользователя на событие со статусом `200 OK`.

- **`/users/{userId}/requests/allWithStatus/list?eventIds={eventIds}&status={status}`** — Получение всех запросов по списку событий и статусу.
    - Принимает параметры:
        - `userId` (`Long`) — идентификатор пользователя.
        - `eventIds` (`List<Long>`) — список идентификаторов событий.
        - `status` (`RequestStatus`) — статус запроса.
    - Возвращает список запросов на участие со статусом `200 OK`.

**POST:**
- **`/users/{userId}/requests?eventId={eventId}`** — Создание нового запроса на участие в событии.
    - Принимает параметры:
        - `userId` (`Long`) — идентификатор пользователя.
        - `eventId` (`Long`) — идентификатор события.
    - Возвращает созданный запрос со статусом `201 Created`.
    - Зависит от:
        - `UserServiceFeign.getUser()` — для получения данных пользователя.
        - `EventServiceFeign.getEventByIdWithoutHttp()` — для проверки существования события.

- **`/users/{userId}/requests/{eventId}/requests`** — Обновление статуса запросов на участие в событии.
    - Принимает параметры пути:
        - `userId` (`Long`) — идентификатор пользователя.
        - `eventId` (`Long`) — идентификатор события.
    - Принимает тело запроса (`EventRequestStatusUpdateRequest`).
    - Возвращает результат обновления статусов со статусом `200 OK`.
    - Зависит от `EventServiceFeign.getEventByIdWithoutHttp()` — для проверки существования события.

**PATCH:**
- **`/users/{userId}/requests/{requestId}/cancel`** — Отмена запроса на участие.
    - Принимает параметры пути:
        - `userId` (`Long`) — идентификатор пользователя.
        - `requestId` (`Long`) — идентификатор запроса.
    - Возвращает отменённый запрос со статусом `200 OK`.

## Микросервис event-service

### Управление ивентами. Администрирование

#### Зависимости от Feign-клиентов event-admin

Контроллер использует следующие Feign-клиенты для взаимодействия с микросервисами:

**UserServiceFeign** (`user-service`):
- `GET /admin/users/allUsersById` — получение списка пользователей по идентификаторам
- `GET /admin/users/{userId}` — получение пользователя по идентификатору

**RequestServiceFeign** (`request-service`):
- `GET /users/{userId}/requests/allWithStatus/list?eventIds={eventIds}&status={status}` — получение запросов по списку событий и статусу

---

#### Реализуемые эндпоинты event-admin

**GET:**
- **`/admin/events`** — Админский поиск событий с фильтрацией и пагинацией.
    - Принимает параметры:
        - `users` (опционально, список `Long`) — идентификаторы инициаторов событий.
        - `states` (опционально, список `EventState`) — статусы событий.
        - `categories` (опционально, список `Long`) — идентификаторы категорий.
        - `rangeStart` (опционально, `LocalDateTime`, формат: `yyyy-MM-dd HH:mm:ss`) — начало диапазона дат.
        - `rangeEnd` (опционально, `LocalDateTime`, формат: `yyyy-MM-dd HH:mm:ss`) — конец диапазона дат.
        - `from` (по умолчанию `0`) — смещение для пагинации.
        - `size` (по умолчанию `10`) — количество записей на страницу.
    - Возвращает список событий со статусом `200 OK`.
    - Зависит от:
        - `UserServiceFeign.getAllUsersById()` — для получения данных инициаторов событий.
        - `RequestServiceFeign.getAllByEventIdInAndStatus()` — для получения количества подтверждённых запросов на каждое событие.

**PATCH:**
- **`/admin/events/{eventId}`** — Админское обновление события.
    - Принимает параметр пути:
        - `eventId` (`Long`) — идентификатор события.
    - Принимает тело запроса (`UpdateEventAdminRequest`).
    - Возвращает обновлённое событие со статусом `200 OK`.
    - Зависит от:
        - `UserServiceFeign.getUser()` — для получения данных инициатора события.
        - `RequestServiceFeign.getAllByEventIdInAndStatus()` — для получения количества подтверждённых запросов на событие.

### Управление ивентами. Общие

#### Реализуемые эндпоинты event-public

**GET:**
- **`/events`** — Публичный поиск событий с фильтрацией, сортировкой и пагинацией.
    - Принимает параметры:
        - `text` (опционально, `String`) — текст для поиска в названии и описании события.
        - `categories` (опционально, список `Long`) — идентификаторы категорий.
        - `paid` (опционально, `Boolean`) — фильтр по платности события.
        - `rangeStart` (опционально, `LocalDateTime`, формат: `yyyy-MM-dd HH:mm:ss`) — начало диапазона дат.
        - `rangeEnd` (опционально, `LocalDateTime`, формат: `yyyy-MM-dd HH:mm:ss`) — конец диапазона дат.
        - `onlyAvailable` (по умолчанию `false`) — фильтр только для доступных событий.
        - `sort` (опционально, `String`) — способ сортировки (например, по дате или просмотрам).
        - `from` (по умолчанию `0`) — смещение для пагинации.
        - `size` (по умолчанию `10`) — количество записей на страницу.
    - Принимает `HttpServletRequest` для записи статистики просмотров.
    - Возвращает список событий со статусом `200 OK`.

- **`/events/{id}`** — Получение события по идентификатору с учётом статистики просмотров.
    - Принимает параметр пути:
        - `id` (`Long`) — идентификатор события.
    - Принимает `HttpServletRequest` для записи статистики просмотров.
    - Возвращает событие со статусом `200 OK`.

- **`/events/{id}/WithoutHttp`** — Получение события по идентификатору без учёта статистики просмотров (для внутреннего использования через Feign).
    - Принимает параметр пути:
        - `id` (`Long`) — идентификатор события.
    - Возвращает событие со статусом `200 OK`.
